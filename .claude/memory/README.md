# 教訓メモリの運用ルール

運用中に得た教訓（ハマったこと・環境の癖・再発しうる失敗）をここに蓄積する。
スキル（手順書）に昇格させるほどではないが、忘れると同じ失敗を繰り返す知識の置き場。

## 書き方のルール（5つだけ）

1. **1ファイル1件**。複数の教訓を1ファイルに詰めない
2. **ファイル名**: `YYYY-MM-DD-<内容が分かる英語スラッグ>.md`（日付は教訓を得た日）
3. **冒頭1行目は必ず1行サマリー**。一覧表示（`head -1 .claude/memory/*.md`）だけで
   全教訓を把握できるようにするため
4. **間違いと判明したら削除する**。古い教訓を残すことより、誤った教訓を消すことが大事。
   削除はコミットメッセージに理由を書けば躊躇しなくてよい
5. **新規作成の前に既存メモを確認**（下のコマンド）。同じ話題があれば新ファイルを
   作らず既存ファイルを更新する（重複が最悪。矛盾する2枚のメモは両方信用できなくなる）

## 使い方

**作業を始める前に**（特にエラーに遭遇したとき）:

```bash
head -1 .claude/memory/*.md   # 全教訓の1行サマリーを一覧
```

関係ありそうなものがあれば本文を読む。

**作業でハマって解決したら**: TEMPLATE.md をコピーして1件書く。
書くべきかの判断基準は「1か月後の別モデルが同じ状況に遭遇したとき、
このメモがあれば30分節約できるか」。YESなら書く。

## テンプレート

`TEMPLATE.md` をコピーして使う。構成:

```
<1行サマリー: 状況→正しい対処 を1文で>

## 状況
（どんな作業中に何が起きたか。エラーメッセージは実物を貼る）

## 誤った対処（やりがちなこと）
（このメモがなければやってしまう間違い）

## 正しい対処
（具体的なコマンド・手順）

## 出典
（いつ・どのセッション/コミット/Actions実行で確認したか）
```
