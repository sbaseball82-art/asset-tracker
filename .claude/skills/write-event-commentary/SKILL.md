# イベント解説の手書き差し込み

`fetch_events.py` は `events_manual.json` に **`detail` が1つでも入っていると、モード設定に
かかわらず自動取得より優先**してその内容を使う（fetch_events.py の main 冒頭で早期リターン）。

## ⚠️ 最重要ルール: 使い終わったら detail を空に戻す

`fetch_events.py` は `events_manual.json` の `date` フィールドを**見ていない**。
`detail` を埋めたまま放置すると、翌日以降も同じ古い解説がスライドに載り続ける。
そのため、解説を書く依頼を受けたら**必ず当日限りの内容であることを確認**し、
ユーザーには「明日以降は自動取得に戻したければ detail を空に戻す（このスキルで『解説をリセットして』
と頼めばよい）」と伝える。「リセットして」と言われたら全 events の `detail` を `""` に戻す。

## 締切

GitHub Actions は **平日 JST 6:30**（cron `30 21 * * 1-5` UTC）に実行される。
それより前に main へプッシュされていなければその日のスライドに反映されない。
現在時刻が 6:30 JST 直前（30分以内）の場合はその旨を警告する。

## 手順

### 1. 素材を集める

ユーザーが解説内容を指定していればそれを使う。「書いておいて」と任された場合は、
当日の米国市場の主要イベント（指数の動き・経済指標・大型個別株ニュース）から3件選ぶ。

### 2. events_manual.json を書く

フォーマット（フィールドは fetch_events.py の仕様に準拠）:

```json
{
  "date": "2026-07-03",
  "events": [
    {"title": "20字以内の日本語見出し", "detail": "60字程度の平易な解説", "dir": "up", "cat": "指数"}
  ]
}
```

- `title`: **20字以内**の日本語。超えるとスライドで見切れる
- `detail`: 60字程度。専門用語を避けた平易な日本語
- `dir`: `"up"` / `"down"` / `"flat"` のみ（スライドの ▲▼ 表示に使う）
- `cat`: `"指数"` / `"マクロ"` / `"個別"` のみ
- events は3件。3件未満でも動く（不足分は "—" で埋められる）が、原則3件書く
- `date` は当日（JST）を入れる。動作には影響しないが記録として正しく保つ

### 3. 検証する（完了条件）

```bash
python3 -c "import json; json.load(open('events_manual.json'))"   # valid JSON
python3 fetch_events.py    # 出力に「✅ events_manual.json を使用」が出ること
```

2つ目のチェックは手動ファイル優先の早期リターンで動くためネット接続不要。
「✅ events_manual.json を使用」が出ない場合は `detail` が全て空のままになっている。

### 4. コミット・プッシュする

締切があるため、検証が通ったら即 main へプッシュする（このリポジトリの日次運用ファイルは
main 直コミットが慣例。作業ブランチ運用中はその指示に従う）。

```
Write event commentary for <YYYY-MM-DD>
```

## 例

**良い例**:

```json
{"title": "FOMC、金利据え置き", "detail": "市場予想どおり政策金利を維持。年内利下げ観測は後退し長期金利は小幅上昇。", "dir": "flat", "cat": "マクロ"}
```

**悪い例** — `"title": "FOMCが政策金利を据え置き、パウエル議長は慎重姿勢"`（20字超で見切れる）。
**悪い例** — `"dir": "上昇"`（`up`/`down`/`flat` 以外は ▲▼ 表示が壊れる。日本語不可）。
**悪い例** — 昨日書いた `detail` を残したまま今日の依頼を「もう書いてあるから不要」と判断する
（date は見られないので昨日の解説が今日も載る。中身が当日のものか必ず確認する）。
