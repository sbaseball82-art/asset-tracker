# .claude/ ディレクトリの歩き方 — 後任モデルへの引き継ぎ書

このファイルは、このリポジトリで作業するAIモデル（あなた）向けの入口。
**作業を始める前にこの1ファイルだけ読めば、どこに何があるか分かる**ように書いてある。

## このリポジトリには2つの独立したシステムがある

| システム | 場所 | 実行時刻 | 内容 |
|---|---|---|---|
| 個人資産スライド | リポジトリ直下（fetch_prices.py 等） | 平日 JST 6:30 | 保有ETF・投信の資産推移を1枚のスライド画像に |
| 日米市場レポート | `stock-daily/` | 毎日 JST 7:30 | 市場全体の指数・値上がりTOP5・ニュースを1枚の画像に |

どちらも GitHub Actions で全自動運用（`.github/workflows/fetch.yml` と `daily_report.yml`）。
オーナーは毎朝生成された画像をXに手動投稿する。**あなたの日常業務は「自動運用が
失敗したときの対応」と「オーナーからの変更依頼の反映」**で、正常時にやることはない。

## 作業の流れ（毎回これに従う）

0. **最新化**: `git fetch origin main` を最初に実行する。あなたのローカルcheckoutは
   依頼時点より古いコミットで止まっていることがある。「data.jsonの日付が古い」
   「今日の生成物がない」と見えても、origin/main には既にあるかもしれない
1. **教訓を確認**: `head -1 .claude/memory/*.md` で過去のハマりどころを一覧する。
   エラーに遭遇したときも、修理を始める前にまずここを見る
2. **スキルを選ぶ**: 依頼内容が下の表に当てはまるなら、必ず該当スキルの手順に従う
3. **作業後にセルフチェック**: `.claude/checklists.md` の該当工程を全部実行する
4. **新しくハマって解決したら**: `.claude/memory/` に教訓を1件残す（書式は memory/README.md）

## スキル一覧（.claude/skills/）

| 依頼の例 | 使うスキル |
|---|---|
| 「◯◯を△円/△口 買った」「売った」 | update-holdings |
| 「新しいETF/投信を追加して」 | add-instrument |
| 「今朝のイベント解説を書いて/差し替えて」 | write-event-commentary |
| 「今日の分を手動で回して」「Actionsが失敗した」 | run-daily-pipeline |
| 「総資産の数字がおかしい」「前日比が異常」 | repair-data-history |

スキルは「ゴール・境界線・検証方法」の形式で書いてある。手順を丸暗記するのではなく、
**ゴールと完了条件を満たすこと**が仕事。ただし各スキルの「禁止事項」だけは
解釈の余地なく文字どおり守ること。

## 絶対に守る3つの境界線（事故が重い順）

1. **`data.json` を削除・再生成しない**。`history` は運用開始から蓄積した唯一の記録で
   再取得不能。異常があっても repair-data-history スキルの手順以外で触らない
2. **API課金を発生させない**。`EVENT_MODE="auto"` への変更と ANTHROPIC_API_KEY の
   設定・使用はオーナーの明示指示があるときのみ
3. **動いているコードを「環境のせいのエラー」で修理しない**。ローカルで失敗しても
   本番Actionsでは毎朝動いている。エラーを見たらまず memory/ を確認し、
   環境問題かコード問題かを切り分けてから動く（切り分け方は run-daily-pipeline スキル）

## ブランチ運用

- 日次生成物（data.json, events.json, slide/, stock-daily/output/）と
  events_manual.json → main へ直接コミット（Actionsと同じ扱い。JST 6:30 / 7:30 が締切）
- コード・スキル・ワークフローの変更 → 作業ブランチ + PR。main への force-push は禁止

## 検証の考え方

「実行が成功した」ではなく「**成果物が正しい**」を完了条件にする。
チェックリストの機械チェック（PASS/FAIL）を通し、画像は必ず目視する。
このリポジトリの最終成果物は毎朝Xに投稿される画像1枚 — 変な文言・欠けた数字・
文字化けがそのまま公開されることを想像して確認すること。
